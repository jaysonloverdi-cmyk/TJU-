# 两天，一个搜题引擎的诞生

## 起因

军理考试逼近。三十道选择题，五分钟限时——你不可能一边翻 PDF 一边搜百度一边在卷子上勾答案。

市面上当然有搜题工具——拍张照，AI 告诉你选什么。但用过的人都知道，它们犯的错误五花八门：把隔壁题的答案安到这道题上，文字陷阱全踩，判断题正反不分，有时干脆瞎编一个。

"我自己也搜得到，"你说，"但我不信它。"

于是我们动手了。

## 第一天：从一堆 if-else 到模块化引擎

起点是一个 242 行的 Python 脚本——`itihey_search.py`。它已经能跑了，27 题对的 30 题，但你清楚这只是"能跑"，离"可靠"还差得远。

下午两点，我们拆掉了这 242 行。

三个小时，单文件变成了七个模块：`search.py` 管 API 搜索和 SQLite 缓存，`ocr.py` 管图片提取，`fallback.py` 管课本和本地题库兜底，`verify.py` 管答案核验，`output.py` 管 Markdown 表格生成，`main.py` 把它们串起来。配置文件从代码里抽出来变成了 `config.yaml`——换一门课，改一行路径就好。

但真正的挑战不在这里。

## 岔路：OCR 的十五个回合

你发来测试 2 的图片——一张 700×15400 像素的长截图，密密麻麻 30 道题。

我用 EasyOCR 跑。第一遍，缺了 7 道题的选项。第二遍加对比度增强，选项粘连。第三遍、第四遍……八分钟过去了，选项还是黏在一起。你说："你用的是千问的视觉模型吗？"

不是。我在用 OCR，而你 vault 根目录里躺着一个 `vision.js`——调千问 VL 模型，90 秒，30 题全部识别，一字不差。我花了十分钟跟 EasyOCR 死磕，而正确工具就在文件夹里，我甚至没有抬头看一眼。

这是整个项目最重要的一课：有时候最快的方法不是把手里的事做得更好，而是换一把更合适的工具。

## 深渊：为什么标了"题库"还会错

Pipeline 跑完了，结果出来——27 道标着 "题库 95%"，但有几道明显是错的。

"开创导弹打导弹先河的导弹"，题库返回 "萨姆-2 地空导弹"。这道题的选项是 A.爱国者 B.飞毛腿 C.响尾蛇 D.战斧。"萨姆-2"根本不在选项里。

发生了什么？itihey 的 API 模糊匹配到了另一道题——那道题里确实有个"萨姆-2"——然后把隔壁题的答案安了过来。API 理直气壮地标了 `use_ai: false`："我从题库找到的。"但它找到的不是这道题。

于是我们加了 `⚠️匹错` 标记——"题库"不再是一个万能通行证。答案在选项里找不到？自动降级。全角括号 `（）` 和半角 `()` 同一个问题可能匹配到不同答案？去括号优先搜。

## 第三天：三个 AI，一张终版表

Pipeline 能搜，但不能验证。你说："我把题目贴给 DeepSeek、Gemini、豆包，你帮我对比。"

于是 `crosscheck.py` 诞生了。

它把一个细活变成了自动化——从四个来源（itihey 题库 + 三个 AI）提取答案，归一化成可比较格式，逐题投票：

- 🟢 四票一致 → 98% 置信，直接采用
- 🟡 题库和 AI 打架，但 AI 内部一致 → 采用 AI 共识
- 🔴 四方混战 → 标红，让人裁决

跑测试 5，30 道题，26 道 🟢，3 道 🟡，1 道 🔴。Q29 那道被标 🟡 的题——"东海防空安全区"——三个 AI 全判对，只有题库说错。你一看：文字陷阱，"安全区"应该是"识别区"。题库是对的，AI 全踩了坑。

这就是为什么不能只信一个源。

## 那些只有 debug 才会发现的 bug

凌晨，API 余额用完了。所有题全标 ❌，一查——402 Payment Required。

全角括号 `（）` 被 `\w` 正则匹配保留，导致搜索变体里永远带着括号，同一道题搜到不同的题库条目。修了两小时才发现，就是六行代码的事。

`found` 命中了题库，但忘了赋给 `best`，第二轮搜索结果被丢了。一个字，一个变量名。

多选答案 "战略支援部队 + 海军 + 空军" 归一化成字母时，"战略支援部队" 6 个字压死了 "海军" 2 个字，归一化函数把短选项全杀了。

这些 bug 没有一个"聪明"——都是那些回头看一眼就觉得荒谬的错误。但它们真实地发生了，也真实地被修掉了。软件工程不是一蹴而就，是每次踩坑之后、每行 fix commit 之后，让事情悄悄变好一点点。

## 最终形态

两天后，从一张裸图到终版答案的完整管线：

```
📷 图片 → 🤖 千问VL识图 → 📝 校对稿 → 👀 你校对+贴AI
                                        ↓
                                ⚡ pipeline 搜 API 题库
                                        ↓
                    🔀 六源交叉验证（itihey+DS+GM+DB+wby+课本）
                                        ↓
                                   📊 终版答案
```

五个测试文件夹，120 题新题库，一份实测数据——平均 87% 的 API 题库命中率，加上三 AI 交叉验证把有效覆盖率推到接近 100%。

这不是一个"AI 帮你做题"的工具。AI 会犯错误——它会把 "萨姆-2"硬塞进 "爱国者"的选项里，会读不出 "安全区"和"识别区"的区别，会在否定题里答反。

这些坑，我们一个一个地标出来、兜住、修正。不是靠更聪明的 AI，而是靠更多的眼睛——题库的、DeepSeek 的、Gemini 的、豆包的、课本的、本地题库的、你的。

## 后记

两天。一个想法、一套代码、一个 GitHub 仓库、一场考试。

如果你也在为军理考试翻 PDF、搜题目、在几十个标签页之间切换——希望这个工具能帮你省下那些时间。也希望你知道，你在试卷上勾下的每一个答案背后，有六个来源在帮你确认。

那不是盲目，那是有据可依。
