# 军理搜题工作流 v3

## 完整流程（严格执行）

```
1. 用户放图片到 测试N/题目/ 
   → 千问VL识别题号 → 重命名图片 → 生成校对稿（题目校对.md）
   
2. 【等待用户校对】用户改校对稿，同时贴给三个AI

3. 用户说"跑"
   → pipeline 调 itihey API → 生成 答案.md
   → crosscheck 六源交叉验证 → 生成 终版答案.md
   
4. 用户逐题审核终版答案
   → 有修改 → 用户告诉我 → 我回：题号/题干/选项/你的修改/建议/判断
   → 确认 → 入新题库
```

## 文件结构

```
测试N/
├── 题目/               ← 测试N-1.jpg ~ 测试N-30.jpg
├── AI回答/             ← 测试N DeepSeek.md / Gemini.md / 豆包.md
├── 题目校对.md          ← 千问VL生成，用户校对
├── 答案.md              ← pipeline API输出
└── 终版答案.md          ← 六源交叉验证，考场用
```

参考模板：`Z军理考试/模拟/`

## 判定规则

| | 条件 | 置信 | 
|--|------|------|
| 🟢 | 题库 + 3AI 全票 | 98% |
| 🟡 | 题库 vs AI分歧，AI内部一致 → 采用AI共识 | 85% |
| 🔴 | 各方不一致 | 60% |

## 关键规则

- **校对稿生成后不立即跑pipeline，等用户说"跑"**
- 🟢 不查本地库/课本，🟡🔴 才查
- 否定题（不是/不属于/不包括）题库AI分歧 → 升级🔴
- 匹错（答案不在选项中）→ 标记 ⚠️匹错，不标题库
- 全角括号 `（）` 搜索时优先去括号
- 豆包粘贴自动排版（拆单行/去空行）
- 校对稿标题格式：`### Q1`（不标类型），`^Q1` 挂在末选项行尾
- 终版答案双链：`[[Z军理考试/测试/测试N/题目校对#Q1]]`
